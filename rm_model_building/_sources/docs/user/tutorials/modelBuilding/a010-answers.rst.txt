   | NOTE: Actual values depend on previous fitting and they may differ
     from the ones shown in this appendix.

   .. container::
      :name: table:refmac_question_7

      .. table:: :math:`Refmac` results. stands for after :math:`Coot`.

         ============== ======= ====== ============ ======
         :math:`Refmac`                :math:`Coot` 
         Statistic      Initial Final  Initial      Final
         \              0.4869  0.4855 0.4971       0.4825
         \              0.0176  0.0212 0.0136       0.0193
         \              1.9186  2.3549 1.8053       0.2382
         \              0.1112  0.1055 0.1470       0.1043
         ============== ======= ====== ============ ======

   | Starting and final values seem to be worse when we do not generate
     a mask volume around the atomic structure when runs both after
     :math:`Coot` + (compare with ) and after :math:`Coot` (compare with
     ). Whitout using a delimiting mask, the whole volume is considered,
     even if the structure fits to a small part of the volume. The use
     of mask is thus especially indicated when map and model show
     different sizes. However, no differences are detected when the
     volume generated by the protocol or normalized volume generated by
     :math:`Coot` are used (data not shown).
